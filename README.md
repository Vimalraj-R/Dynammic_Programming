🚀 Mastering Dynamic Programming (DP)Welcome to my Dynamic Programming repository! This repository serves as a centralized hub for tracking my journey, solutions, and architectural breakdowns of classic algorithmic problems solved using optimization techniques.🧠 The Universal 4-Step DP FrameworkWhenever I encounter a new DP problem, I systematically break it down from scratch using this exact mental pipeline instead of guessing the solution:Define the "State" (dp[i]): Identify exactly what the array/table indices represent relative to the final problem goal.Identify the "Last Decision": Determine the very last independent choice or transition step available at index i.Connect Present to Past (The Transition Formula): Translate the choices mathematically into a recurrence relation using $\min()$, $\max()$, or summation ($\sum$).Establish the Base Cases: Pinpoint the foundational inputs (like dp[0] or dp[1]) that have instantly trivial, non-calculated answers.📂 Repository Structural LayoutEvery folder in this repository follows a clean, highly scannable documentation layout designed for quick technical review:Section 1: Problem Description & Corner Testcases.Section 2: Architectural Explanation & High-level Pseudocode.Section 3: Production-ready Optimized Source Code ($O(N)$ Space/Time variants where applicable).🗺️ Core Dynamic Programming ArchetypesThis repository partitions problems into their distinct sub-families based on choice variations and structural constraints:🎒 1. 0/1 Knapsack & DecisionsThe Rule: You are given items with unique weights/values. For each item, you make a binary choice: 0 (Skip) or 1 (Take). Items cannot be reused.Core Transition:$$\text{dp}[i] = \max(\text{dp}[i-1], \text{dp}[i-2] + \text{value})$$Classic Examples: House Robber, Subset Sum, Partition Equal Subset Sum.🔄 2. Unbounded Knapsack / IterativeThe Rule: You have a fixed array of choices and a target capacity, but you have an infinite supply of each choice. You can pick the same element repeatedly.Core Transition:$$\text{dp}[i] = \min(\text{dp}[i], \text{dp}[i - \text{choice}] + 1)$$Classic Examples: Coin Change (Min Coins), Combination Sum IV, Rod Cutting.🗺️ 3. Grid / 2D PathingThe Rule: Moving across an $M \times N$ spatial matrix under strict directional restrictions (e.g., only move Down or Right), creating a one-way directed acyclic path graph.Core Transition:$$\text{dp}[r][c] = \text{cost}[r][c] + \min(\text{dp}[r-1][c], \text{dp}[r][c-1])$$Classic Examples: Unique Paths, Minimum Path Sum, Dungeon Game.🧵 4. Sequence Alignment / StringsThe Rule: Comparing subsequences or substrings across multiple string variables, checking character matching choices across index dimensions.Core Transition:$$\text{If Match: } \text{dp}[i-1][j-1] + 1 \quad \text{Else: } \max(\text{dp}[i-1][j], \text{dp}[i][j-1])$$Classic Examples: Longest Common Subsequence (LCS), Edit Distance, Longest Palindromic Substring.📈 Optimization TrackerProblem NameFamilyTime ComplexitySpace ComplexityStatusCoin ChangeUnbounded Knapsack$O(N \times T)$$O(T)$✅ SolvedUnique PathsGrid / 2D Paths$O(M \times N)$$O(N)$⏳ OptimizingHouse Robber0/1 Knapsack$O(N)$$O(1)$✅ Solved
